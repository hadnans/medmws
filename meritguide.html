What is MERiT?

Understanding the toolbox architecture and imaging pipeline

🧠

Core Purpose

MERiT (Microwave Radar-based Imaging Toolbox) is a free, open-source MATLAB framework developed by EMFMed for reconstructing radar-based images in medical settings. Its primary application is detecting tumors in breast tissue using ultra-wideband (UWB) microwave pulses transmitted and received by antenna arrays surrounding the breast.

The core physical principle: cancerous tissue has significantly different dielectric properties than healthy fatty breast tissue. Microwave pulses reflect differently off tumors, and these scattered signals are computationally focused into an image.

🏗️

Input Data

What files to prepare, their formats, and how to load them

📋

Required Input Files

MERiT requires four types of data provided as CSV files or MATLAB arrays:

File / VariableDescriptionDimensionsUnits
frequencies.csvList of measurement frequencies (UWB sweep)F × 1 vectorHz (e.g., 1e9 to 9e9)
antenna_locations.csv3D Cartesian coordinates of each antenna phase centerN × 3 matrix [x, y, z]Meters

💡
You need at least two scan files captured at different antenna rotations (e.g., 0° and 36°) to perform rotational subtraction artefact removal, which cancels the dominant skin reflection and isolates the tumor response.
📂

Loading Data Manually

%% Load each CSV file individually
frequencies = dlmread('example_data/frequencies.csv');
% F×1 vector of frequency values in Hz

antenna_locations = dlmread('example_data/antenna_locations.csv');
% N×3 matrix: [x y z] per antenna, in meters

channel_names = dlmread('example_data/channel_names.csv');
% C×2 matrix: [Tx_index Rx_index] per channel

scan1 = dlmread('example_data/B0_P3_p000.csv');
% C×F complex matrix: scan at rotation 0°

scan2 = dlmread('example_data/B0_P3_p036.csv');
% C×F complex matrix: scan at rotation 36°

🔄

Artefact Removal: Rotational Subtraction

Raw microwave signals are dominated by the strong skin-surface reflection. Rotational subtraction cancels it:

%% Isolate tumor response by rotational subtraction
% Scans at 0° and 36° have nearly identical skin reflections.
% Subtraction cancels the skin artefact, leaving the tumor signal.

signals = scan1 - scan2;
% signals: C×F matrix with artefact suppressed
⚠️
Other artefact removal methods (time-gating, frequency-domain filtering) can also be applied before passing signals to the beamformer. MERiT is agnostic to the method used.

Signal Management

Functions in merit.signals — filtering, reordering, and selecting channels

📡

Purpose

Before beamforming, you often need to filter or reorganize your signal set. The merit.signals sub-package provides utilities to:

  • Keep only monostatic channels (Tx = Rx, same antenna transmits and receives)
  • Keep only bistatic channels (Tx ≠ Rx, different antennas)
  • Exclude specific channels that are noisy or faulty
  • Reorder antenna index numbering for consistency
🔧

Signal Management Functions

get_monostatic — Keep same-antenna channels only

%% Keep only monostatic channels (Tx == Rx)
[signals_mono, channels_mono] = merit.signals.get_monostatic( ...
    signals, channel_names);

% signals_mono  → reduced C'×F matrix
% channels_mono → C'×2 index list of kept channels
💡
Monostatic mode is simpler to implement in hardware. Bistatic captures more spatial diversity and generally yields better image quality.

get_bistatic — Keep cross-antenna channels only

%% Keep only bistatic channels (Tx ~= Rx)
[signals_bi, channels_bi] = merit.signals.get_bistatic( ...
    signals, channel_names);
% Filters out self-coupling (diagonal channels where Tx==Rx)

exclude_channels — Remove specific faulty channels

%% Exclude noisy or faulty channels by row index
bad_channels = [3, 7, 12];
[signals_clean, channels_clean] = merit.signals.exclude_channels( ...
    signals, channel_names, bad_channels);
% Removes specified rows from signals and channel_names

reorder — Remap antenna indices

%% Reorder antenna numbering
% Useful when hardware labels differ from expected ordering
new_order = [2,1,4,3,6,5,8,7];
channel_names_reordered = merit.signals.reorder(channel_names, new_order);
% Remaps all Tx/Rx indices in channel_names accordingly

Imaging Domain

Functions in merit.domain — defining the 3D reconstruction grid

🌎

What is the Imaging Domain?

The imaging domain is a discrete 3D grid of points inside the breast volume. The DAS beamformer computes a reconstruction value at every grid point — essentially asking: "How much reflected signal energy arrives from this location?" High energy at a point indicates a strong reflector (a potential tumor).

Image resolution is directly determined by the domain's point spacing (resolution) and its shape and size (radius).

📐

Domain Constructor Functions

merit.domain.hemisphere — Standard breast shape

%% Create a hemispherical imaging domain
[points, axes_] = merit.domain.hemisphere( ...
    'radius', 7e-2, ...        % 7 cm breast radius
    'resolution', 2.5e-3);   % 2.5 mm grid spacing

% points → M×3 matrix: [x y z] of every grid point (meters)
% axes_  → cell array {x_axis, y_axis, z_axis} for plotting
⚠️
Finer resolution increases image quality but dramatically increases computation time. Use 5 mm for fast prototyping and 2.5 mm for final results.

merit.domain.cylinder — Cylindrical domain

%% Cylindrical domain (for some experimental setups)
[points, axes_] = merit.domain.cylinder( ...
    'radius', 6e-2, ...
    'height', 8e-2, ...
    'resolution', 2.5e-3);

Output Variables

OutputTypeDescription
pointsM × 3 doubleEach row is a 3D [x,y,z] coordinate in meters of a grid point inside the domain
axes_Cell {1×3}Unique x, y, z axis vectors used by get_slice for 2D axis labeling
DAS Beamforming

Functions in merit.beamform — computing propagation delays and reconstructing the image

🧮

How DAS Works

Delay-and-Sum (DAS) is a synthetic aperture beamforming algorithm. For each grid point p in the imaging domain, it:

  1. Calculates the two-way travel time from each Tx antenna to point p and back to each Rx antenna
  2. Phase-shifts (delays) each channel's signal by that travel time in the frequency domain
  3. Coherently sums all phase-aligned signals

When the grid point coincides with a real reflector (tumor), all delayed signals align constructively and produce a high-amplitude sum. Elsewhere, signals cancel destructively. The absolute value of the sum becomes the image intensity at that point.

⏱️

Step 1: merit.beamform.get_delays

Returns a delay function handle that, given a set of domain points, computes the two-way propagation delay for every Tx/Rx antenna pair.

%% Compute propagation delay function
delays = merit.beamform.get_delays( ...
    channel_names, ...             % C×2 [Tx, Rx] channel list
    antenna_locations, ...         % N×3 antenna positions in meters
    'relative_permittivity', 8);   % ε_r of breast medium

% 'delays' is a function handle.
% Call delays(points) to get a C×M delay matrix (seconds)
% for M domain points and C channels.
ParameterTypeDescription
channel_namesC × 2 intTx-Rx antenna index pairs; determines which signal paths are used
antenna_locationsN × 3 double3D positions of all antennas in meters; used to compute Euclidean path lengths
'relative_permittivity'scalar doubleEffective dielectric constant of the medium. Affects wave speed: v = c / sqrt(ε_r). Use ~9 for fatty breast, ~8 for coupling fluid.
⚠️
Choosing the correct relative_permittivity is critical. An incorrect value shifts all delay calculations, blurring or mislocating the tumor. For heterogeneous phantoms, use the effective average permittivity of the breast medium.
🖼️

Step 2: merit.beamform — Run the Beamformer

%% Perform DAS image reconstruction
img = abs(merit.beamform( ...
    signals, ...               % C×F complex signal matrix
    frequencies, ...           % F×1 frequency vector in Hz
    points, ...                % M×3 domain grid points
    delays, ...                % delay function from get_delays()
    merit.beamformers.DAS));   % specify DAS algorithm

% img → M×1 vector of image intensities (one per domain point)
% abs() gives the real-valued magnitude image

%% Normalize for display
img_norm = img / max(img);
💡
The raw beamformer output is complex-valued. Always apply abs() to get the magnitude image. Normalize with img / max(img) for consistent display across scans.

Available Beamformers

BeamformerDescription
merit.beamformers.DASStandard Delay-and-Sum — coherent summation of delayed signals. Fast, widely used, good baseline performance.
merit.beamformers.DMASDelay-Multiply-and-Sum — multiplies signal pairs before summing. Reduces clutter at the cost of higher computation.

GPU and Parallel Acceleration

%% GPU acceleration (just pass gpuArray data)
signals_gpu = gpuArray(signals);
frequencies_gpu = gpuArray(frequencies);
img_gpu = abs(merit.beamform(signals_gpu, frequencies_gpu, points, delays, ...
    merit.beamformers.DAS));
img = gather(img_gpu);  % bring result back to CPU

%% Parallel pool (MATLAB Parallel Computing Toolbox)
parpool();  % MERiT auto-detects the pool — no code changes needed

Image Quality Metrics

%% Signal-to-Clutter Ratio (SCR)
% Measures how much brighter the tumor peak is vs. background
scr = merit.metrics.scr(img, points, tumor_location, tumour_radius);
fprintf('SCR: %.2f dB

Visualization

Functions in merit.visualize — 2D slices and 3D volume rendering

🖼️

2D Slice Visualization

To display a 2D image, extract a planar cross-section from the 3D reconstruction using merit.visualize.get_slice.

merit.visualize.get_slice — Extract a 2D plane

%% Extract an axial (XY) slice at a specific Z depth
im_slice = merit.visualize.get_slice( ...
    img, ...         % M×1 intensity vector
    points, ...      % M×3 domain grid points
    axes_, ...       % cell array from merit.domain
    'z', 35e-3);     % z-coordinate of slice (35 mm depth)

% im_slice → 2D matrix (Nx × Ny), ready for imagesc()

figure();
imagesc(axes_{1}*1000, axes_{2}*1000, im_slice);
axis image; colormap hot; colorbar;
xlabel('X (mm)'); ylabel('Y (mm)');
title('Axial Slice at z = 35 mm');

All Three Orthogonal Orientations

%% Axial   — XY plane at constant Z (most common)
slice_xy = merit.visualize.get_slice(img, points, axes_, 'z', 35e-3);

%% Sagittal — XZ plane at constant Y
slice_xz = merit.visualize.get_slice(img, points, axes_, 'y', 0);

%% Coronal  — YZ plane at constant X
slice_yz = merit.visualize.get_slice(img, points, axes_, 'x', 0);

%% Display all three orthogonal views side by side
figure('Name', 'Orthogonal Views');
subplot(1,3,1); imagesc(slice_xy); colormap hot; title('Axial (XY)'); axis image;
subplot(1,3,2); imagesc(slice_xz); colormap hot; title('Sagittal (XZ)'); axis image;
subplot(1,3,3); imagesc(slice_yz); colormap hot; title('Coronal (YZ)'); axis image;
💡
Use the hot or parula colormap — high-intensity regions (potential tumors) appear bright white/yellow against a dark background, making them easy to spot.
🤩

3D Volume Visualization

To inspect the full 3D structure of the reconstruction, reshape the intensity vector into a 3D grid and use MATLAB volumetric rendering.

Method 1: Isosurface rendering

%% Reshape flat vector into 3D volume
Nx = length(axes_{1}); Ny = length(axes_{2}); Nz = length(axes_{3});
img_vol = reshape(img / max(img), [Nx, Ny, Nz]);

%% Render isosurface at 50% of peak intensity
figure();
fv = isosurface(img_vol, 0.5);
patch(fv, 'FaceColor', 'red', 'EdgeColor', 'none', 'FaceAlpha', 0.6);
daspect([1 1 1]); view(3);
camlight; lighting phong;
xlabel('X'); ylabel('Y'); zlabel('Z');
title('3D Isosurface — Reconstructed Image');

Method 2: Multi-slice stacking in 3D

%% Display multiple Z-slices stacked in 3D space
figure();
for k = 1:5:Nz
    z_val = axes_{3}(k);
    slice_k = merit.visualize.get_slice(img, points, axes_, 'z', z_val);
    [X, Y] = meshgrid(axes_{1}, axes_{2});
    Z_plane = z_val * ones(size(X));
    surf(X*1000, Y*1000, Z_plane*1000, slice_k', ...
        'EdgeColor', 'none', 'FaceAlpha', 0.4);
    hold on;
end
colormap hot; colorbar; view(3); axis tight;
xlabel('X (mm)'); ylabel('Y (mm)'); zlabel('Z (mm)');
title('3D Multi-Slice Volumetric View');

Method 3: scatter3 point cloud

%% Show only high-intensity points as a 3D scatter
threshold = 0.3 * max(img);
idx = img > threshold;

figure();
scatter3(points(idx,1)*1000, points(idx,2)*1000, points(idx,3)*1000, ...
         20, img(idx), 'filled');
colormap hot; colorbar;
xlabel('X (mm)'); ylabel('Y (mm)'); zlabel('Z (mm)');
title('3D Point Cloud — High-Intensity Regions');
Complete Workflow Example

A full end-to-end MATLAB script from data loading to 3D visualization

🚀

Full Pipeline Script

%% ================================================
%% MERIT Complete Imaging Pipeline
%% Breast Cancer Detection with DAS Beamforming
%% ================================================

%% STEP 1: Add MERIT to MATLAB path
addpath('/path/to/MERIT');

%% STEP 2: Load input data
frequencies       = dlmread('example_data/frequencies.csv');
antenna_locations = dlmread('example_data/antenna_locations.csv');
channel_names     = dlmread('example_data/channel_names.csv');
scan1             = dlmread('example_data/B0_P3_p000.csv');
scan2             = dlmread('example_data/B0_P3_p036.csv');

%% STEP 3: Artefact removal
signals = scan1 - scan2;

%% STEP 4: Signal management (keep bistatic channels)
[signals, channel_names] = merit.signals.get_bistatic(signals, channel_names);

%% STEP 5: Define hemispherical imaging domain
[points, axes_] = merit.domain.hemisphere( ...
    'radius', 7e-2, ...
    'resolution', 2.5e-3);

%% STEP 6: Compute propagation delays
delays = merit.beamform.get_delays( ...
    channel_names, antenna_locations, ...
    'relative_permittivity', 8);

%% STEP 7: Run DAS beamforming
img = abs(merit.beamform( ...
    signals, frequencies, points, delays, ...
    merit.beamformers.DAS));
img_norm = img / max(img);

%% STEP 8a: 2D axial slice at z = 35 mm
im_slice = merit.visualize.get_slice(img_norm, points, axes_, 'z', 35e-3);
figure('Name', '2D Axial Slice', 'Color', 'k');
imagesc(axes_{1}*1000, axes_{2}*1000, im_slice);
axis image; colormap hot; colorbar;
xlabel('X (mm)'); ylabel('Y (mm)');
title('DAS Reconstruction — Axial Slice (z=35mm)');

%% STEP 8b: 3D isosurface rendering
Nx = length(axes_{1}); Ny = length(axes_{2}); Nz = length(axes_{3});
img_vol = reshape(img_norm, [Nx, Ny, Nz]);
figure('Name', '3D Reconstruction');
fv = isosurface(img_vol, 0.5);
patch(fv, 'FaceColor', 'red', 'EdgeColor', 'none', 'FaceAlpha', 0.7);
daspect([1 1 1]); view(3); camlight; lighting phong;
title('3D Isosurface — Potential Tumor Location');

%% STEP 9: Image quality metrics
tumor_xyz = [0, 20e-3, 35e-3];   % known tumor location [x,y,z] meters
tumor_r   = 5e-3;                 % tumor radius (5 mm)
scr = merit.metrics.scr(img, points, tumor_xyz, tumor_r);
fprintf('Signal-to-Clutter Ratio: %.2f dB
', 20*log10(scr));
✅

Quick Tips

  • Always call abs() on beamformer output before visualization
  • Normalize: img / max(img) for consistent colormaps
  • Use colormap hot or parula for clearest tumor contrast
  • Start with resolution = 5e-3 for speed, refine to 2.5e-3
  • Tune relative_permittivity for your specific phantom material
  • Call parpool() before beamforming to enable parallel speedup
  • Compare SCR before and after signal management to assess improvement
