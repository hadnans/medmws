Section 01

Introduction & Purpose

MERIT (Microwave Emerging and Radar-based Imaging Toolbox) is an open-source MATLAB toolbox developed primarily at University College Dublin (UCD) by the research group of Prof. Martin O'Halloran, Dr. Emily Porter, and collaborators. Its fundamental goal is to provide a modular, reproducible, and extendable platform for microwave radar breast imaging — an active area of research targeting non-ionizing, non-contact tumor detection as an alternative or complement to X-ray mammography.

The toolbox abstracts the full imaging pipeline: from loading raw antenna S-parameter measurements, through calibration and signal conditioning, to 3-D image reconstruction via beamforming algorithms and final image quality evaluation. Because everything is encapsulated inside a MATLAB package namespace (+merit), it integrates cleanly into any existing codebase without polluting the global function space.

Why MERIT Was Built

Prior to MERIT, researchers had to re-implement beamformers from scratch for each study, making results difficult to compare across groups. MERIT solves this by providing:

  • Algorithm parity — identical implementations of DAS, DMAS, and FD-DAS across all experiments.
  • Standardized data structures — the channels struct and points array are universal across all functions.
  • Plug-in extensibility — new algorithms, metrics, and preprocessing functions are added with minimal boilerplate.
  • Reproducibility — the toolbox is versioned and published, enabling citation and comparison.
ℹ
Research Context

MERIT is specifically designed for confocal microwave imaging (CMI) where an antenna array illuminates a breast phantom or tissue, and backscattered signals are processed to localize dielectric anomalies (tumors). The toolbox is not limited to breast imaging — any near-field radar scenario with multi-static or mono-static array data can use it.

Section 03

MATLAB Package System Explained

MERIT uses MATLAB package folders (directories prefixed with +). This is a namespacing mechanism similar to Python's module system. Understanding it is essential before modifying the toolbox.

How Packages Work

Any folder named +name becomes the namespace name. Functions inside it are called as name.function(). Nested packages (e.g., +merit/+algorithms/das.m) are called as merit.algorithms.das().

% Calling a packaged function — three equivalent approaches:

% 1. Full namespace (always works, most explicit)
img = merit.algorithms.das(channels, points, freq);

% 2. Import into local scope (convenient for scripts)
import merit.algorithms.*;
img = das(channels, points, freq);

% 3. Store as function handle (useful for passing as parameter)
alg = @merit.algorithms.das;
img = alg(channels, points, freq);
MATLAB
⚠
Common Mistake

If you create a new file inside +merit/+algorithms/ but MATLAB cannot find it, the most common cause is that the parent of +merit is not on the path. Run which merit to confirm — if it returns the path to merit.m, the root is on the path and your new function should be accessible as merit.algorithms.yourfunction().

Section 04

The channels Data Structure

The channels struct is the central data object in MERIT. Everything flows through it. Understanding every field is mandatory before building new functionality. It represents all measurement data collected from the antenna array.

channels ← main measurement data struct │ ├── .antennas [N × 3] double ← (x,y,z) positions of all N antennas [meters] │ ├── .time [1 × T] double ← time vector [seconds] │ Δt = 1/(2 × f_max) typical │ ├── .freq [1 × F] double ← frequency vector [Hz] (for FD algorithms) │ ├── .data [N × N × T] double ← calibrated time-domain signals │ data(i,j,:) = signal from tx i → rx j │ diagonal = monostatic (i==j) │ ├── .fd_data [N × N × F] complex ← freq-domain signals (if FD preprocessing done) │ ├── .speed [scalar] double ← effective propagation speed [m/s] │ typically c/√ε_r for homogeneous medium │ e.g. 0.7×c for breast phantom │ └── .mask [N × N] logical ← which tx-rx pairs to include true = use, false = skip often excludes adjacent pairs (near-field)

Building a channels Struct from Scratch

%% Example: Build a channels struct for a 16-element circular array
N       = 16;
r       = 0.07;                    % array radius 7 cm
theta   = linspace(0, 2*pi, N+1); theta(end) = [];

% Antenna positions on a circle at z = 0
channels.antennas = [r*cos(theta)', r*sin(theta)', zeros(N,1)];

% Time vector (2 ns window, sampled at 0.5 GHz bandwidth)
dt                = 1e-12;         % 1 ps step
channels.time    = (0:dt:2e-9); % 0 to 2 ns

% Effective propagation speed (coupling medium ε_r ≈ 10)
channels.speed   = 3e8 / sqrt(10);

% Load S21 data — shape must be [N x N x length(time)]
channels.data    = raw_s21_data;   % your measured/simulated data

% Mask: use all pairs except self-coupling (diagonal)
channels.mask    = ~eye(N, N, 'logical');

The channels.speed value directly determines how time delays are translated into spatial distances. A 10% error in speed propagates as a ~10% positioning error in the reconstructed image. For a heterogeneous breast, use a frequency-averaged effective permittivity. For FDTD simulation data, use the exact permittivity of the coupling liquid.

Section 05

The Imaging Domain (points)

The imaging domain is a 3-D voxel grid over which the beamformer evaluates its focusing function. Every voxel becomes one pixel in the final image. The domain is passed to all algorithms as a Px3 matrix — where P is the total number of grid points and each row is the [x, y, z] coordinate of one voxel in meters.

Creating a Domain with merit.domain.get_domain

% Define 3-D imaging region: a 14×14×6 cm volume
x_range = [-0.07, 0.07];   % [m]
y_range = [-0.07, 0.07];
z_range = [0.00, 0.06];
res     = 0.002;             % 2 mm resolution

points = merit.domain.get_domain(x_range, y_range, z_range, res);
% points is [P × 3] where P = 71×71×31 = 156,401 voxels

% Internally, get_domain does:
%   [X,Y,Z] = meshgrid(x_range(1):res:x_range(2), ...);
%   points  = [X(:), Y(:), Z(:)];

% Compute grid dimensions from domain parameters

%% Annotated internal logic of merit.m (simplified)

function image = merit(channels, points, algorithm, varargin)

  % ── Step 1: Parse optional name-value arguments ──────────
  p = inputParser();
  p.addParameter('FreqRange',    [],      @isnumeric);
  p.addParameter('Window',       [],      @isnumeric);
  p.addParameter('Apodization',  'hann', @ischar);
  p.addParameter('Speed',        [],      @isscalar);
  p.parse(varargin{:});
  opts = p.Results;

  % ── Step 2: Resolve algorithm to function handle ─────────
  alg_fn = merit.algorithms.get_algorithm(algorithm);

  % ── Step 3: Apply preprocessing ──────────────────────────
  if ~isempty(opts.FreqRange)
    channels = merit.process.filter(channels, opts.FreqRange);
  end
  if ~isempty(opts.Window)
    channels = merit.process.time_window(channels, opts.Window);
  end
  if ~isempty(opts.Speed)
    channels.speed = opts.Speed;
  end

  % ── Step 4: Build settings struct for algorithm ──────────
  settings.apodization = opts.Apodization;

  % ── Step 5: Call the algorithm ───────────────────────────
  image = alg_fn(channels, points, settings);

end

Pro Tip: Bypass the Wrapper

For batch processing (many phantoms), calling merit() re-runs inputParser and preprocessing every time. Pre-compute the filtered channels once, then call the algorithm function directly: merit.algorithms.das(channels_filtered, points, settings) — this can be 2–5× faster in loops.

Calibration — merit.process.calibrate

Calibration (also called subtraction-based artifact removal) removes the dominant early-time reflections from the skin interface and antenna coupling. Without it, the skin reflection is typically 30–60 dB stronger than the tumor response and completely masks it. 

% merit.process.calibrate removes artifact using an average reference
function ch_cal = calibrate(channels_tumor, channels_ref)
  %
  % Method 1: Direct subtraction (if reference scan exists)
  %   ch_cal.data = channels_tumor.data - channels_ref.data;
  %
  % Method 2: Average subtraction (no explicit reference)
  %   For each antenna i, the reference for channel (i,j) is
  %   the average of all other channels involving antenna i:
  %
  %   ref_i = mean(data(i, :, :), 2)   <- mean over all Rx for this Tx
  %   ch_cal.data(i,j,:) = data(i,j,:) - ref_i
  %
  
  if nargin == 2
    ch_cal = channels_tumor;
    ch_cal.data = channels_tumor.data - channels_ref.data;
  else
    ch_cal = channels_tumor;
    for i = 1:size(channels_tumor.data,1)
      ref = mean(channels_tumor.data(i,:,:), 2);
      ch_cal.data(i,:,:) = channels_tumor.data(i,:,:) - ref;
    end
  end
end

Bandpass Filtering — merit.process.filter

Applied in the time domain using a zero-phase Butterworth filter. Removes out-of-band noise and limits the frequency content to the usable range of the antenna system. 

% Apply 2–8 GHz bandpass filter to all channels
freq_range = [2e9, 8e9];  % Hz
channels   = merit.process.filter(channels, freq_range);

%% Internally, filter.m does:
%   fs     = 1 / (channels.time(2) - channels.time(1));  % sample rate
%   Wn     = freq_range / (fs/2);                         % normalized
%   [b,a]  = butter(order, Wn, 'bandpass');
%   For each tx-rx pair: data(i,j,:) = filtfilt(b,a, squeeze(data(i,j,:)))
%   filtfilt → zero-phase (no phase distortion)
MATLAB
💡
Filter Order Selection

MERIT defaults to a Butterworth order of 3. Higher orders (5–7) give steeper roll-off but introduce more Gibbs-like ringing at sharp transitions. For UWB signals spanning 1–10 GHz, order 3–4 is usually optimal. If you see oscillatory artifacts in early-time signals post-filtering, reduce the order.

Time Windowing — merit.process.time_window

A Hann (or rectangular) window function multiplied onto each time-domain signal. It gates the signals to include only echoes returning from within the imaging domain, discarding late-arriving multipath signals. 

% Gate signals to include echoes from 0.5–2.0 ns (returns from <30 cm)
t_window = [0.5e-9, 2.0e-9];  % seconds
channels  = merit.process.time_window(channels, t_window);

%% How to compute the correct window bounds:
%  t_min = 2 * d_min / speed;   % closest antenna-to-domain edge distance
%  t_max = 2 * d_max / speed;   % furthest imaging point distance

d_min  = 0.005;                  % 5 mm (antenna offset)
d_max  = sqrt(0.07^2 + 0.06^2); % corner of domain
t_min  = 2*d_min  / channels.speed;
t_max  = 2*d_max  / channels.speed;

Delay Calculation — merit.beamform.delay

The delay calculation is the computational heart of every time-domain beamforming algorithm. For each voxel in the imaging domain, it computes the two-way travel time for a wavefront emitted from transmitting antenna i, reflecting off the voxel at position p, and received at antenna j.

τij(p) = (|p − ri| + |p − rj|) / veff
Round-trip propagation delay for tx antenna i → point p → rx antenna j
%% merit.beamform.delay — annotated implementation
function delays = delay(channels, points)
  % channels.antennas : [N × 3]  antenna positions
  % channels.speed    : scalar   effective propagation speed
  % points            : [P × 3]  imaging grid voxel coords
  % delays            : [N × N × P]  two-way delay in seconds

  N = size(channels.antennas, 1);
  P = size(points, 1);
  v = channels.speed;

  % Distance from each antenna to each voxel: [N × P]
  for k = 1:N
    diff = points - channels.antennas(k,:);       % [P × 3] broadcast
    dist(k,:) = sqrt(sum(diff.^2, 2))';          % [1 × P]
  end

  % Two-way delay: tx i distance + rx j distance, divided by speed
  for i = 1:N
    for j = 1:N
      delays(i,j,:) = (dist(i,:) + dist(j,:)) / v;  % [1 × P]
    end
  end
end
MATLAB
⚠
Memory Warning for Large Domains

The delays array has shape [N × N × P]. For a 16-antenna array on a 2 mm grid (156k voxels): 16×16×156,401 × 8 bytes ≈ 320 MB. For 32 antennas at 1 mm: over 6 GB. MERIT handles this by computing and using delays chunk-by-chunk in the algorithm functions rather than pre-allocating the full tensor.

Section 10

DAS — Delay-and-Sum Algorithm

Delay-and-Sum (DAS) is the foundational beamforming algorithm — conceptually simple but highly effective for initial imaging. It works by aligning all received echoes to the same reference time (the propagation delay to a candidate voxel) and then coherently summing them. Voxels coinciding with a scatterer accumulate constructively; all other voxels partially cancel.

IDAS(p) = | Σi,j ∈ M wij · sij(τij(p)) |²
DAS image intensity at voxel p — where M is the active antenna pair mask, w are apodization weights, and s is the time-domain signal

Step-by-Step Internal Logic

function image = das(channels, points, settings)
  N      = size(channels.antennas, 1);
  P      = size(points, 1);
  image  = zeros(P, 1);

  % Get apodization (spatial weighting) for all active pairs
  weights = merit.beamform.apodization(channels, settings);

  % Loop over all active transmit-receive pairs
  for i = 1:N
    for j = 1:N
      if ~channels.mask(i,j), continue; end

      % Extract signal for this tx-rx pair: [T × 1]
      sig = squeeze(channels.data(i,j,:));

      % Compute round-trip delay from tx i → point p → rx j: [P × 1]
      r_tx = vecnorm(points - channels.antennas(i,:), 2, 2);
      r_rx = vecnorm(points - channels.antennas(j,:), 2, 2);
      tau  = (r_tx + r_rx) / channels.speed;   % [P × 1] seconds

      % Interpolate signal at each delay (sub-sample accuracy)
      % interp1 with 'linear' or 'spline' — spline is more accurate
      sig_delayed = interp1(channels.time, sig, tau, 'linear', 0);

      % Accumulate weighted contribution: [P × 1]
      image = image + weights(i,j) * sig_delayed;
    end
  end

  % Square to get power (envelope detection)
  image = image .^ 2;
end

ℹ
DAS Limitations

DAS produces the image with the highest resolution among MERIT's algorithms but also the highest clutter level (sidelobes). Coherent grating lobes appear when inter-element spacing exceeds λ/2. For breast imaging at 6 GHz center frequency in tissue (λ ≈ 10 mm), this constrains element spacing to < 5 mm for full coverage.

Section 11

DMAS — Delay-Multiply-and-Sum

DMAS is an enhanced beamformer that replaces the simple coherent summation of DAS with a pairwise multiplication of delayed signals before summation. This cross-correlation operation gives it superior clutter rejection at the cost of resolution (the effective beam pattern is the square of the DAS beam). It was introduced specifically for microwave breast imaging by Lim et al. (2008) and is one of MERIT's most frequently cited algorithms.

IDMAS(p) = Σi<j Σk<l, (k,l)≠(i,j) sij(τij) · skl(τkl)
DMAS: multiply all pairs of delayed signals before summing — effectively computing cross-correlation at zero-lag

DMAS vs DAS — Key Differences

PropertyDASDMAS
Mathematical operationLinear sumSum of pairwise products
Effective beam patternOne-way beamSquare of DAS beam
SCR improvement over DASBaselineTypically +5 to +15 dB
Computational complexityO(N²·P)O(N⁴·P) — much slower
Resolution (−3 dB beamwidth)Sharper~40% wider
Clutter artifactsHigherLower
 
function image = dmas(channels, points, settings)
  N      = size(channels.antennas, 1);
  P      = size(points, 1);
  image  = zeros(P, 1);

  % First, compute all delayed signals: store in cell array [N×N]
  sig_del = cell(N, N);
  for i = 1:N
    for j = 1:N
      if ~channels.mask(i,j), continue; end
      sig = squeeze(channels.data(i,j,:));
      tau = (vecnorm(points-channels.antennas(i,:),2,2) + ...
             vecnorm(points-channels.antennas(j,:),2,2)) / channels.speed;
      sig_del{i,j} = interp1(channels.time, sig, tau, 'linear', 0);
    end
  end

  % Get list of active pairs
  [ri, rj] = find(channels.mask);
  M = numel(ri);

  % DMAS: multiply each unique pair of active channels
  for m = 1:M
    for n = m+1:M           % n > m ensures no double-counting
      a = sig_del{ri(m), rj(m)};  % [P × 1] delayed signal m
      b = sig_del{ri(n), rj(n)};  % [P × 1] delayed signal n
      % Signed sqrt multiplication preserves sign while reducing squaring
      image = image + sign(a.*b) .* sqrt(abs(a.*b));
    end
  end
end
MATLAB
⚠
Performance Warning

DMAS has O(M²) complexity where M is the number of active antenna pairs. For a 16-antenna full multi-static array (M = 240 pairs), the inner double-loop runs 28,680 iterations per image. Pre-allocate the delayed signals and consider GPU acceleration (gpuArray) for real-time applications.

Section 12

Frequency-Domain DAS (fd_das.m)

Frequency-Domain DAS (FD-DAS) operates on the complex frequency-domain S-parameter data directly, without converting to time domain first. It applies a phase shift corresponding to the propagation delay, then sums — equivalent to coherent time-domain DAS, but often faster for frequency-sparse data.

IFD-DAS(p) = | Σi,j ∈ M Σf wij · S̃ij(f) · ej2πf·τij(p) |²
FD-DAS: phase-shift and coherently sum in frequency domain — equivalent to time-domain convolution with a matched filter
function image = fd_das(channels, points, settings)
  N    = size(channels.antennas, 1);
  P    = size(points, 1);
  freq = channels.freq;          % [1 × F]
  image = zeros(P, 1);

  for i = 1:N
    for j = 1:N
      if ~channels.mask(i,j), continue; end

      % Frequency-domain signal for this pair: [1 × F]
      S = squeeze(channels.fd_data(i,j,:))';

      % Round-trip delay for all voxels: [P × 1]
      tau = (vecnorm(points-channels.antennas(i,:),2,2) + ...
             vecnorm(points-channels.antennas(j,:),2,2)) / channels.speed;

      % Phase steering matrix: [P × F]
      % Each row: e^(j*2*pi*f*tau_p) for a single voxel across all freqs
      phase = exp(1j * 2*pi * tau * freq);  % broadcasting: [P × F]

      % Multiply by spectrum and sum over frequencies: [P × 1]
      % This is equivalent to evaluating IFFT at delay time tau
      contribution = real(sum(phase .* S, 2));

      image = image + contribution;
    end
  end

  image = image .^ 2;
end

Section 13

Algorithm Dispatcher — get_algorithm.m

The dispatcher is a simple but crucial piece of the architecture. It translates a string name to a function handle, enabling the merit() wrapper to work generically and allowing users to pass algorithm names as strings rather than remembering full namespace paths. 

function alg_fn = get_algorithm(algorithm)
% Resolve algorithm specification to a callable function handle.

  if isa(algorithm, 'function_handle')
    % Already a function handle — use directly
    alg_fn = algorithm;

  elseif ischar(algorithm) || isstring(algorithm)
    % String → look up in registry
    switch lower(char(algorithm))
      case 'das'
        alg_fn = @merit.algorithms.das;
      case 'dmas'
        alg_fn = @merit.algorithms.dmas;
      case 'fd_das'
        alg_fn = @merit.algorithms.fd_das;
      otherwise
        error('merit:get_algorithm:unknown', ...
              'Unknown algorithm: "%s". Valid: das, dmas, fd_das', algorithm);
    end

  else
    error('merit:get_algorithm:invalid', ...
          'Algorithm must be a string or function handle.');
  end
end

Image Quality Metrics

MERIT provides three standard metrics for quantitatively evaluating reconstructed images. These are the same metrics used in virtually all published microwave breast imaging papers, enabling direct comparison with literature.

1. Signal-to-Clutter Ratio (SCR) — merit.metrics.scr

SCR = 10 log₁₀ ( maxp∈Ωt I(p) / meanp∈Ωc I(p) )
SCR: ratio of peak intensity in tumor region to mean intensity in clutter region [dB]

tumor_loc = [0.02, 0.01, 0.03];   % [x,y,z] in meters
r_tumor   = 0.01;                   % 1 cm search radius

% Internally:
%   dist_to_tumor = vecnorm(points - tumor_loc, 2, 2);
%   tumor_mask    = dist_to_tumor <= r_tumor;          % Ωt

Section 17

Adding New Algorithms to MERIT

MERIT's architecture makes adding a new beamforming algorithm straightforward. Follow this exact procedure to ensure compatibility with all existing toolbox functions, including the dispatcher, metrics, and merit() wrapper.

1

Create the function file

Create a new .m file inside +merit/+algorithms/. Name it after your algorithm. The file must follow the standard signature exactly.

2

Implement the signature contract

Your function must accept (channels, points, settings) and return a P×1 double image vector. Deviating from this contract will break compatibility with merit() and the metrics pipeline.

3

Register in the dispatcher

Add a case entry in get_algorithm.m mapping your algorithm's string name to its function handle. This is optional — you can always pass the function handle directly — but it provides a cleaner API.

4

Test against the demo data

Run your algorithm against data/example_scan.mat and compare SCR, SMR, and LE against DAS baseline to validate correctness.

Example: Implementing a Custom Weighted DAS (WDAS)

%% File: +merit/+algorithms/wdas.m
%% Weighted Delay-and-Sum with distance-based attenuation compensation

function image = wdas(channels, points, settings)
%WDAS  Weighted Delay-and-Sum with attenuation compensation.
%
%  Extends DAS by multiplying each sample by r² (geometric spreading
%  compensation) before summation. Improves imaging of deep targets.
%
%  Specific settings fields used:
%    settings.alpha  - attenuation coefficient [Np/m], default = 20
%

  % Default settings
  if ~isfield(settings, 'alpha'), settings.alpha = 20; end

  N      = size(channels.antennas, 1);
  P      = size(points, 1);
  image  = zeros(P, 1);
  v      = channels.speed;
  alpha  = settings.alpha;

  for i = 1:N
    for j = 1:N
      if ~channels.mask(i,j), continue; end
      sig = squeeze(channels.data(i,j,:));

      r_tx  = vecnorm(points - channels.antennas(i,:), 2, 2);
      r_rx  = vecnorm(points - channels.antennas(j,:), 2, 2);
      tau   = (r_tx + r_rx) / v;

      % Attenuation + geometric spreading compensation weight
      path_len = r_tx + r_rx;
      w_atten  = path_len .* exp(alpha * path_len);   % [P × 1]

      sig_delayed = interp1(channels.time, sig, tau, 'linear', 0);
      image = image + w_atten .* sig_delayed;
    end
  end

  image = image .^ 2;
end

Adding New Image Quality Metrics

Custom metrics live in +merit/+metrics/. They receive the flat image vector and points array, and return a scalar value.

Example: Half-Power Beamwidth (HPBW) Metric

%% File: +merit/+metrics/hpbw.m
%% Compute 3-dB half-power beamwidth around the image peak

function bw_mm = hpbw(image, points, axis_dir)
%HPBW  Half-power beamwidth of the peak response along a given axis.
%
%  bw_mm = hpbw(image, points, 'x')  % measure width along x-axis
%  axis_dir: 'x' | 'y' | 'z'
%

  % Find peak voxel
  [~, idx_peak] = max(image);
  peak_pos = points(idx_peak, :);

  % Select axis index
  ax_map = struct('x',1, 'y',2, 'z',3);
  col = ax_map.(axis_dir);

  % Extract 1D profile along chosen axis through peak
  tol   = 1e-4;   % 0.1 mm tolerance for "same line"
  other_axes = setdiff([1,2,3], col);
  on_axis = all(abs(points(:, other_axes) - peak_pos(other_axes)) < tol, 2);
  profile_img = image(on_axis);
  profile_ax  = points(on_axis, col);

  % Normalize and find -3 dB crossing points
  profile_n = profile_img / max(profile_img);
  half_power_thresh = 0.5;    % -3 dB

  above = profile_n >= half_power_thresh;
  idx_start = find(above, 1, 'first');
  idx_end   = find(above, 1, 'last');
  bw_mm     = (profile_ax(idx_end) - profile_ax(idx_start)) * 1000;
end

Adding Custom Preprocessing Functions

Preprocessing functions in MERIT follow a transform pattern: they take a channels struct, modify its .data (or .fd_data) field, and return the modified struct. This makes them composable and chainable.

Example: Rotation Correction (Array Alignment)

%% File: +merit/+process/rotate_array.m
%% Correct for rotational misalignment of the antenna array

function channels_out = rotate_array(channels, theta_deg)
%ROTATE_ARRAY  Rotate all antenna positions around z-axis by theta_deg.
%
%  Useful for correcting scan-to-scan rotational misregistration
%  in rotational array systems.

  theta = theta_deg * pi / 180;
  Rz    = [cos(theta), -sin(theta), 0; ...
           sin(theta),  cos(theta), 0; ...
           0,           0,          1];

  channels_out              = channels;
  channels_out.antennas    = (Rz * channels.antennas')';
end

Example: Entropy-Based Artifact Removal

%% +merit/+process/entropy_filter.m
%% Remove channels whose time-domain signal has anomalously high entropy
%% (suggests coupling failure or RF interference)

function channels_out = entropy_filter(channels, threshold)
  if nargin < 2, threshold = 3.0; end

  channels_out = channels;
  N = size(channels.mask, 1);
  removed_count = 0;

  for i = 1:N
    for j = 1:N
      if ~channels.mask(i,j), continue; end
      sig = squeeze(channels.data(i,j,:));
      sig_norm = abs(sig) / sum(abs(sig));
      H = -sum(sig_norm .* log2(sig_norm + eps));  % Shannon entropy
      if H > threshold
        channels_out.mask(i,j) = false;
        removed_count = removed_count + 1;
      end
    end
  end
  fprintf('entropy_filter: masked %d anomalous channels\n', removed_count);
end

Tips, Hints & Best Practices

🔬 Research Workflow Summary

  • Always calibrate first — the skin artifact is 30–60 dB above the tumor signal; no algorithm can compensate for it.
  • Set channels.speed from the coupling medium permittivity, not free space. Error here is the #1 cause of localization failures.
  • Use DAS first for rapid prototyping — it's the fastest to compute and easiest to debug. Switch to DMAS once the pipeline is working.
  • Always visualize the channels.data B-scan (antenna index vs. time) before running beamforming to catch preprocessing issues early.
  • Normalize your image by its maximum value before computing SCR/SMR to ensure consistent comparison across algorithms.
  • For publication, report SCR, SMR, and LE for all algorithms on the same phantom.

Common Bugs and Fixes

🐛 Image is all zeros or NaN

Cause: channels.speed is too large, so all delays exceed the time window length. interp1 returns 0 (the fill value) for out-of-range delays.

Fix: Check that max(tau) ≤ channels.time(end) for any voxel. Print max(max(tau)) and compare to channels.time(end).

max_tau = max(vecnorm(points,2,2)) * 2 / channels.speed;
fprintf('Max delay: %.2f ns | Time window end: %.2f ns\n', max_tau*1e9, channels.time(end)*1e9);
🐛 Image peak appears at array center (not tumor location)

Cause: Calibration has not been applied — the large early-time skin/coupling artifact focuses at the geometric center of the array.

Fix: Apply merit.process.calibrate before beamforming. Verify calibration worked by checking that the B-scan no longer shows a strong coherent early arrival.

🐛 MATLAB says "merit.algorithms.myalgo not found"

Cause: Either (a) the root folder is not on the MATLAB path, or (b) the file is named differently from the function name, or (c) the file is in the wrong folder.

Fix: Run which merit — must return path to merit.m. Run exist('merit.algorithms.myalgo', 'file') — must return 2. Ensure filename exactly matches function name.

🐛 FD-DAS image is much worse than TD-DAS

Cause: channels.fd_data was generated from a truncated time window or with incorrect FFT normalization, causing phase distortion.

Fix: Ensure the IFFT/FFT pair uses consistent normalization. If converting from time domain: use fft(data, [], 3) with no normalization factor, and the corresponding IFFT uses ifft(fd_data, [], 3) / T. Verify by round-tripping a known signal.
